😎 triRoute

Es un helper creado para evitar escribir rutas o urls en las vistas, controladores o clases personalizadas.

Para que pueda ser usado en los proyectos de cada cliente en su primer actualización debe ser ejecutado en la terminal el comando de composer después del pull en git:

composer dump-autoload

Seguidamente del comando para limpiar cache de Laravel:

php artisan config:cache

Al ser creado especialmente para tri se debe usar en condicionales if.
Además en esta primera versión solo debe ser usado en condicionales:

OR ó ||

De este modo:

{{-- Ejemplo en blade --}}
@if(triRoute::validateOR('desarrollo') || triRoute::validateOR('vym') || triRoute::validateOR('listos') ... Así sucesivamente)
  {{-- code --}}
@endif

Este helper retorna un valor booleano true ó false.

👀 Uso en vistas (view)

Para el uso en las vistas con blade no es necesario importar el helper basta con llamar la clase y la función a usar.

{{ triRoute::validateOR('nombre_instancia') }}

El parámetro nombre_instancia es el nombre del cliente en sí. Por ejemplo:

{{ triRoute::validateOR('desarrollo') }}

{{ triRoute::validateOR('listos') }}

En condicionales:

@if(triRoute::validateOR('desarrollo') || triRoute::validateOR('vym'))
  {{-- code --}}
@endif

⌨️ Uso en controladores o clases personalizadas

Para el uso en controladores debe importarse al inicio del archivo. Por ejemplo:

<?php
namespace App\Http\Controllers;

use App\Http\Controllers\Controller;

//Se importa
use triRoute;

class ExampleController extends Controller
{
    public function example()
    {
      //Se usa
      if(triRoute::validateOR('desarrollo') || triRoute::validateOR('vym')) {
          //code
        }
    }
}
?>
